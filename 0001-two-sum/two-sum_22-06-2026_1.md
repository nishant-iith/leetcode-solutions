## Solution Code
```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        int n = nums.length;
        for(int i = 0; i < n; i++){
            for(int j = i + 1; j < n; j++){
                if(nums[i] + nums[j] == target) return new int[]{i, j};
            }
        }
        return new int[]{-1, -1};
    }
}
```

## **1. Algorithm Analysis**

The "Two Sum" problem asks: given an array of integers and a target sum, return the indices of the two numbers which add up to the target. If no such pair exists, return `[-1, -1]`.

### **Approach Overview:**

1. **Iterate through each element** in the array.
2. For the current element `nums[i]`, compute the required complement: `complement = target - nums[i]`.
3. **Check if complement exists in the array** from the remaining elements (i.e., not before `i` but before this `i`, to prevent double-checking).
4. If complement is found, return the indices `[i, j]`, where `j` is in `[i+1, ... n-1]`.

> This is the **two-pointer approach** via a sorted array, or a **hash-backed lookup** approach.

### **Time & Space Complexity:**
- **Time Complexity**: O(n log n) due to sorting (if sorting is done first).  
- **Space Complexity**: O(1) if no extra space, except recursion stack, which is O(log n).  
  *However, this algorithm can be optimized to O(n) with a hash set.*
