## Solution Code
```java
import java.util.Stack;

class MinStack {

    Stack<Integer> stack = new Stack<>();
    Stack<Integer> minStack = new Stack<>();

    public MinStack() {
        
    }
    
    public void push(int val) {
        stack.push(val);

        if (minStack.isEmpty() || val <= minStack.peek()) {
            minStack.push(val);
        }
        else {

            minStack.push(minStack.peek());
        }
    }
    
    public void pop() {
        stack.pop();
        minStack.pop();
    }
    
    public int top() {
        return stack.peek();
    }
    
    public int getMin() {
        return minStack.peek();
    }
}
```

# Detailed Analysis: MinStack Implementation

## 1. Algorithm Analysis

### The Core Problem
A standard Stack provides $O(1)$ time for `push`, `pop`, and `peek`. However, finding the minimum element in a standard stack typically requires iterating through all elements, which takes $O(n)$ time. The goal of `MinStack` is to retrieve the minimum element in $O(1)$ time.

### The Strategy: Parallel State Tracking
The provided solution uses a **Two-Stack Approach**. 

1.  **`stack` (The Main Stack):** This stores all the elements pushed into the data structure in the order they arrive.
2.  **`minStack` (The Auxiliary Stack):** This acts as a "history" of the minimum value. At any given index $i$, the value in `minStack` represents the minimum value present in the main `stack` from index $0$ to $i$.

### Logical Flow
*   **`push(val)`**: 
    *   The value is always added to the main `stack`.
    *   To maintain the minimum, the code checks: *Is this new value smaller than or equal to the current minimum?*
        *   **If yes:** Push the new value onto `minStack`.
        *   **If no:** Push the current minimum (the top of `minStack`) again. This ensures that both stacks always stay the same height, maintaining a 1:1 mapping between an element and the minimum value at that point in time.
*   **`pop()`**: 
    *   Since both stacks are synchronized in height, popping from both simultaneously ensures that when an element is removed, its corresponding "minimum at that time" is also removed.
*   **`top()`**: 
    *   Returns the top of the main stack.
*   **`getMin()`**: 
    *   Returns the top of the `minStack`, which by definition is the minimum of all elements currently in the main stack.

## 3. Complexity Analysis

### Time Complexity
| Method | Complexity | Justification |
| :--- | :--- | :--- |
| `push(int val)` | $O(1)$ | Adding an element to a `java.util.Stack` (based on `Vector`) takes constant time. |
| `pop()` | $O(1)$ | Removing the top element from a stack takes constant time. |
| `top()` | $O(1)$ | Accessing the top element via `peek()` takes constant time. |
| `getMin()` | $O(1)$ | Accessing the top of the auxiliary stack takes constant time. |

**Overall Time Complexity: $O(1)$ per operation.**

### Space Complexity
**Complexity: $O(n)$**

**Justification:**
*   We are using two stacks. In the worst-case scenario (e.g., where every pushed element is smaller than the previous one, or even if they are larger), both stacks will grow linearly with the number of elements $n$.
*   Since we store $n$ elements in `stack` and $n$ elements in `minStack`, the total space used is $2n$. In Big-O notation, constants are dropped: $O(2n) \rightarrow O(n)$.
